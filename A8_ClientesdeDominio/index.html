Configurar clientes

Debemos tener en cuenta varios aspectos en la configuración del cliente Lubuntu:

  • La fecha/hora del sistema debe sincronizarse con el PDC.
  • GNU/Linux y PDC, deben estar en la misma red, por lo que es aconsejable configurar la red de las máquinas virtuales en modo "puente".

  • Los serviores deben tener la configuración de red estática.

Configuración de red de los clientes.

Para configurar los DNS en una máquina Debian/Ubuntu, podemos hacer uso del propio fichero de configuración de red /etc/network/interfaces.

Para comprobar si la resolución de nombres (Configuracion de los servidores DNS) está funcionando correctamente hacemos una prueba con el siguiente comando:nslookup www.iespuertodelacruz.es

Unirse al dominio

Vamos a unir el cliente Ubuntu al PDC Windows Server usando una aplicación de entorno gráfica llamada Likewise. Para instalar dicha aplicación debemos copiar los repositorios pasados por los compañeros para que nos aparezca en nuestro sistema operativo y añadirlos al fichero /etc/apt. Una vez hecho esto instalamos el programa Likewise mediante el comando apt-get install likewise-open, y apt-get install likewise-open-gui porque también nos hace falta ese paquete para tenerlo con entorno gráfico.

Una vez instalado el programa, queremos saber dónde está y ejecutarlo para empezar a utilizarlo para unirnos al PDC Windows Server. Ésto se hace a través de los comandos whereis domainjoin-guiy domainjoin-gui.

Luego, completamos la siguiente información de configuración:

  • Dejamos el nombre de equipo como estaba.
  • En "Domain" ponemos el nombre de nuestro PDC.

Finalmente, nos pedirá un usuario y contraseña donde pondremos el administrador del dominio y su contraseña. Luego saldrá una pantalla como que es correcto y se pide reiniciar.

Comprobación

Desde el cliente, entramos al sistema con algún usuario del dominio y comprobamos que funciona correctamente.
Entramos en la consola y mediante el comando whoiam nos muestra qué usuario somos y con el cat /etc/passwd|grep quintero no nos muestra nada porque exactamente no hay usuario en esta máquina denominado "quintero".

Por último, vamos al Windows Server 2008 y entramos en la carpeta "Computers", del Directorio Activo, en la que tendrá que aparecer nuestra máquina GNU/Linux.
